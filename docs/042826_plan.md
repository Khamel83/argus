## Phase 1 - Correctness Fixes

1. Fix SearchAPI status and routing.
   - File: `argus/providers/searchapi.py`
   - Problem: `SearchApiProvider` is a stub and never performs HTTP.
   - Files: `argus/broker/policies.py`, `tests/test_providers.py`, `tests/test_broker.py`
   - Decide either:
     - implement the real SearchAPI adapter and add it to mode preference lists, or
     - remove SearchAPI from provider docs/registration until implemented.
   - Preferred fix: implement adapter, then add `ProviderName.SEARCHAPI` to tier-3 positions in every relevant mode.

2. Fix session query refinement.
   - File: `argus/sessions/refinement.py`
   - File: `argus/broker/session_flow.py`
   - Problem: `refine_query()` assumes current query is already appended, but broker calls it before adding the current query. First real follow-up is not refined.
   - Fix: treat all existing `session.queries` as prior context in the broker path, or make `SessionSearchService` append before refinement and persist after.

3. Fix MCP tool exposure mismatch.
   - File: `argus/mcp/server.py`
   - Problem: `search_health`, `search_budgets`, and `test_provider` are documented as MCP tools but only exposed over stdio.
   - Fix: expose safe/admin tools over authenticated remote MCP, or explicitly document them as stdio-only.

4. Fix extraction completeness behavior or docs.
   - File: `argus/extraction/extractor.py`
   - Problem: completeness is assessed after later paid/archive extractors, but strong truncation does not continue the chain after Jina and later steps.
   - Fix: either apply `_should_continue_for_completeness()` consistently across all steps, or document that paid/archive extractors are terminal after quality pass.

5. Close SSRF redirect gap.
   - File: `argus/extraction/extractor.py`
   - Problem: initial URL is checked, but redirect targets are not revalidated by all local extractors.
   - Fix: disable redirects where possible or validate final URLs before fetching/rendering content.
